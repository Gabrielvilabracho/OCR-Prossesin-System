# Sample Client Documentation

This documentation tree is a sanitized placeholder for a public repository.

The original private delivery documentation was removed because it included
client-specific discovery, runbooks, implementation history, operational notes,
and evaluation artifacts that are not appropriate for public release.

## Public-safe contents

- `technical/CONTRACT.md` — neutral sample backend/frontend contract.
- `architecture/DESIGN.md` — neutral sample architecture overview.
- `operations/RUNBOOK.md` — public-safe operational checklist.
- `ai/evaluations/golden-dataset/README.md` — guidance for synthetic fixtures.

## Rules for future docs

- Use fictional organizations and `example.com` contacts.
- Use synthetic IDs, sample bucket names, and non-production URLs.
- Do not include private stakeholder names, private domains, production runbooks,
  reverse-anonymization mappings, or credential-like values.
- Keep compatibility directory names only until a later rename work unit can
  update imports, scripts, and docs safely.
